Prerequisites


TuxSim runs on intel-based computers running Linux or Mac OS X and that have GCC 4.0 or greater installed. You can install pre-compiled binaries or build TuxSim from the source code. If you wish to build TuxSim yourself, you will need to install the Boost C++ libraries.


Running TuxSim


Run tuxsim from the command line as follows:
Usage: tuxsim [options]* <config_file>

The following is a detailed description of the options used to control Cufflinks:


Arguments:
<config_file> Configure the simulation run with parameters stored in this file
Options:
-h/--help Prints the help message and exits
-A/--abundances <file_name> An abundance file. When this option is specified, TuxSim skips assignment of abundances to sample molecules and proceeds directly to fragmentation.
-F/--fragments <file_name> A fragment output file from a previous simulator run. When this option is specified, TuxSim skips the fragmentation step, and simply performs read sequencing on the specified library.

TuxSim Input


TuxSim takes as input a configuration file specifying what type of assay to simulate along with the parameters that determine various parameters of the assay such as sequencing platform and various protocol steps.

This file is in an INI-like format, where parameters are set with lines of the form:


a.config.group.param_name=value

These parameters can be grouped into sections:


[a.config.group]
param_name_1=value
param_name_2=value
Configuration options:
Group Parameter Value Type Default Description
input fasta_dir Directory path <none> Directory with FASTA files for source molecules
source_pool source_gtf File path <none> GTF file of spliced molecules for sequencing. If this is not set, the source fasta files themselves will be the pool sequences (i.e. resequencing)
fragment_length distribution String gaussian The distribution to use for the lengths of the fragments sequenced during the run. Supported values are: "gaussian", "empirical". For "gaussian" see fragment_length.mean and fragment_length.std_dev. For "empirical", see fragment_length.empirical_dist_file
fragment_length mean Decimal 200.0 Mean length of sequenced fragments. Used only for "gaussian" fragment length distributions.
fragment_length std_dev Decimal 40.0 Standard deviation of the length of sequenced fragments. Used only for "gaussian" fragment length distributions.
fragment_length empirical_dist_file File path <none> A tab-delimited file containing the probability of observing a fragment of each length. NOT IMPLEMENTED
sequencing read_type String illumina_paired Read configuration for the run. Supported values are: "illumina_single", "illumina_paired".
sequencing read_length Integer 76 The length of each read
sequencing num_fragments Integer 20000000 The number of fragments to sequence in the run.
output prefix String <sim> All output files will share this prefix


TuxSim Output


TuxSim produces several output files:


    prefix.sam

    A SAM file of aligned reads


    prefix.srcab

    A table of source molecules and their relative abundances


    prefix_1/2.fq

    FASTQ files of reads (two files are produced for paired end simulation runs).
